Conditional Context Optimization (CoCoOp)

작성자: 여행 초짜
작성일: 2023.11.21

시작하기 앞서 틀린 부분이 있을 수 있으니, 틀린 부분이 있다면 지적해주시면 감사하겠습니다.

이번에 소개할 논문은 Conditional Context Optimization (CoCoOp) 입니다. 기존 CoOp가 가지던 overfitting의 문제점을 해결하기 위해 등장한 모델입니다.

이 논문도 CoOp와 동일한 저자가 저술하였으며, 구조는 CoOp와 동일하지만 image representation을 추가로 받는다는 차이점이 있습니다. CoOp에 대해 잘 모르신다면 CoOp 설명글을 참고하시기 바랍니다.

“ CoCoOp의 동기
Motivation of CoCoOp ”

CoCoOp는 CoOp의 일반화 성능을 지적하고 overfitting 문제점을 해결하고자 동일 저자가 제시한 방법입니다.

  1. Soft prompt를 downstream task에 대해서 적용하다보니 in-class에서는 우수한 성능을 보이지만, unseen class에 대해 일반화가 되지 않음.
  2. 예측하는 데 있어 overfitting이 발생함(위와 같은 맥락).

실제로 아래의 경우처럼, 모델이 학습한 in-class에 대해서는 CoOp와 CoCoOp가 비슷한 성능을 보여주고 CLIP보다 좋은 결과를 보여줍니다. 다만 zero-shot classification을 수행할 때 학습하지 않았던 unseen class에 대해서는 CoOp는 오히려 CLIP보다 낮은 성능을 보이지만 CoCoOp는 CLIP보다 더 나은 일반화 성능을 보여줍니다.

CLIP, CoOp, CoCoOp의 일반화 성능, 출처: CoCoOp 논문


저자가 일반화 성능이 떨어지는 원인으로써 제시한 증거가 바로 CoOp 논문의 nearest neighbor word 결과이고, 이 결과가 상당히 문맥적이지 않으며 text domain을 전혀 이해하지 못한다고 언급합니다. 저자는 이렇게 일반화 성능이 약하고 overfitting이 발생한 이유가 CoOp의 static 설계 때문에 발생했다고 판단하여, dynamic한 soft prompting이 가능하도록 light weight network를 추가하여 학습하는 방법을 제안합니다.

“ CoCoOp의 구조 & 학습
CoCoOp Architecture & Traning ”

1. CoCoOp 구조
CoOp의 구조는 CoOp와 동일합니다. 다만 Meta-Net이라는 새로운 모델이 추가가 됐을 뿐입니다. CoOp의 구조는 CoOp 설명글을 참고하시고, 여기서는 Meta-Net의 구조만 설명하도록 하겠습니다.

저자가 Meta-Net을 도입한 이유는 이 모델을 도입함으로써 기존 text에 붙어서 text encoder로만 들어가던 soft prompting 방식에 image-guide를 더해줄 수 있기 때문입니다. 그리고 Meta-Net이 뭔가 엄청난 모델이 아닌, 단순히 2개의 레이어를 사용한 아주 단순한 모델입니다.

  • Meta-Net: 2-layered MLP (Linear-ReLU-Linear)
    • 기존 prompt를 최적화하는 과정에서 image representation이 약할 수 있던 문제점을 보완.
    • Context overfitting을 줄이기 위해 특정 class에 집중하는 것 대신 input 전체에 초점을 맞출 수 있어서 일반화가 잘 됨.
처음에 저자는 단순히 M개의(sequence-wise) context token을 각각 학습하는 M개의 neural network들을 두자고 했지만, CoOp와 비교하여 너무 무거워져서 context token에 공통으로 더해질 Meta-Net 하나만 구성하는 concept으로 구성했다고 합니다.

2. CoOp 학습
학습 방법도 정말 간단합니다. 심지어 저자들은 pre-training을 직접 수행하지도 않고, fine-tuning만 진행합니다.
  1. Phase 1: CLIP pre-training
  2. Phase 2
    • M x D 크기의 soft prompt를 초기화 (M은 hyperparameter).
    • Meta-Net을 초기화.
    • Pre-trained 모델을 freeze 한 후, soft prompt와 Meta-Net만 학습.

“ CoCoOp 결과
Results of CoCoOp ”

저자들은 CLIP, CoOp, CoCoOp의 3가지 모델에 대해서 in-class (학습 때 사용한 클래스), unseen class (zero-shot)에 대해 실험을 수행하여 성능을 비교합니다. 아래 결과 표에서 Base, New, H는 각각 in-class, unseen class, in-class + unseen class의 결과입니다. 실제로 unseen class의 zero-shot에서는 CoCoOp가 CoOp보다 좋지만 여전히 CLIP이 우세합니다. 다만 저자는 in-class + unseen class의 결과가 CLIP보다 CoCoOp가 더 좋아서 CLIP보다 사용성이 더 좋다고 주장합니다.
