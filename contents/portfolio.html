  • Open-domain Chat System Modeling (2022.03 ~ 2023.04)
    • Description: I conducted a project to build an everyday conversation chatbot model using anonymized Korean data. For generative models, I used DialoGPT, GPT-2, and BART as baselines. Inspired by the CLIP model, I developed a Text-CLIP model utilizing embeddings of sequential text. This model retrieves appropriate responses from a database based on previous sentences.
    • Contribution: This project was conducted by two individuals. I was responsible for preprocessing anonymized SNS data, and I developed the generative baselines using DialoGPT and GPT-2. Additionally, I handled the development of the retrieval model.
    • Tech Stacks: Python, PyTorch, Hugging Face, Uvicorn, Websocket, FastAPI
    • Performance
      • I demonstrated the developed generative chatbot model at a public institution and proceeded with the project by collecting additional chat log data provided by the institution.
      • I implemented token streaming using Uvicorn, WebSocket, and FastAPI in an asynchronous manner, allowing multiple chatbot models to generate sentences simultaneously in real-time.
      • I performed the task of converting data into '해요체' (polite form) through morphological analysis and applied it to the chatbot (Contribution: 50%).

  • Document Extractive Summarization (2022.06 ~ 2022.08)
    • Description: This project was conducted at the request of a public institution. To extract summaries, I trained models and used KoBERT and Sentence BERT (SBERT) as baselines for experimentation. Specifically, I developed a model for extractive summarization using KoBERT. Given the large number of sentences in each document, I employed the sentence n-gram technique to address this challenge.
    • Contribution: This project was conducted by two individuals, and we focused on training the KoBERT model and developing the sentence n-gram technique.
    • Tech Stacks: Python, PyTorch, Hugging Face

  • Language Model Lightweighting (2022.06 ~ 2022.12)
    • Description: During the development of an open-domain chat model, I conducted language model lightweighting to enhance training resource efficiency and scalability for various auxiliary tasks. I achieved lightweighting by sharing weights between the encoder and decoder of the existing seq2seq-based model. Additionally, I designed the model to perform various tasks by simply changing its masks. The resulting model demonstrated exceptional performance in semantic tasks compared to conventional models, and it was versatile for multi-modal applications such as text-text and image-text scenarios.
    • Contribution: I conducted model implementation, training, quantitative and qualitative evaluations. Additionally, a senior researcher performed code reviews.
    • Tech Stacks: Python, PyTorch

  • Text Augmentation using GAN (2022.01 ~ 2022.12)
    • Description: Before the emergence of LLMs, I developed a model for text data augmentation. At that time, text augmentation techniques relied on supervised learning-based autoregressive methods to generate sentences. However, this approach led to data memorization, where the model simply reproduced the training data. To address this issue, I introduced an unsupervised GAN approach to learn the text embedding space and convert it into sentences.
    • Contribution: This project was conducted by two individuals. I handled most of the tasks including model implementation based on the idea, model training, and paper writing. The review of the paper and code was done collaboratively with a senior researcher.
    • Tech Stacks: Python, PyTorch
    • Paper: https://arxiv.org/abs/2306.17181

  • Online Commercial OCR (2021.09 ~ 2021.12)
    • Description: This project evaluated the performance of online advertisement OCR based on publicly available Korean text-in-the-wild and public document OCR data from sources such as AI Hub, and compared the results of the trained model with prominent OCR services like Tesseract and ABBYY.
    • Contribution: I was responsible for all aspects of the project, including data preprocessing with OpenCV and implementing OCR models using CRAFT and transformers.
    • Tech Stacks: Python, OpenCV, PyTorch, Tesseract

  • Document Inverted Indexing (2021.09 ~ 2021.09)
    • Description: I developed an inverted indexing program that considers word frequency and lexicographic order in documents. Using StringBuffer, I implemented the program to generate the inverted index within 3 seconds for 300,000 documents.
    • Contribution: I was responsible for all aspects of the project.
    • Tech Stacks: Java
